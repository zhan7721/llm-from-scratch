# 多智能体系统

> **模块 06 -- 智能体，第 04 章**

多智能体系统允许多个大语言模型支持的智能体协作完成单个智能体难以胜任的复杂任务。不是让一个智能体包揽一切，而是让专业化的智能体各司其职，分享结果，并在彼此的工作基础上继续推进。本模块实现了三种协调模式：顺序流水线、并行执行和多轮辩论。

## 全局视图

多智能体系统协调多个专业智能体共同完成任务：

```
用户任务
    |
    v
MultiAgentSystem（多智能体系统）
    |
    +---> Orchestrator（编排器，协调策略）
    |         |
    |         +---> 顺序模式: 智能体 A -> 智能体 B -> 智能体 C
    |         +---> 并行模式: 智能体 A, 智能体 B, 智能体 C（同时运行）
    |         +---> 辩论模式: 智能体 A <-> 智能体 B <-> 智能体 C（迭代优化）
    |
    +---> SharedMemory（共享内存，智能体间上下文）
    |
    v
最终结果
```

### 核心组件

```
Agent（智能体）
    +-- name: str           （唯一标识符）
    +-- role: str           （如"研究员"、"评审员"）
    +-- goal: str           （如"查找事实信息"）
    +-- llm_fn: Callable    （fn(messages) -> str）

SharedMemory（共享内存）
    +-- store(key, value)   （存储数据）
    +-- retrieve(key)       （加载数据）
    +-- get_all()           （查看所有数据）
    +-- clear()             （清空）

Orchestrator（编排器）
    +-- run_sequential(task)  （链式执行，传递上下文）
    +-- run_parallel(task)    （独立执行，收集结果）
    +-- run_debate(task)      （多轮迭代优化）

MultiAgentSystem（多智能体系统）
    +-- mode: str           （"sequential"、"parallel"、"debate"）
    +-- run(task)           （使用选定模式执行）
```

### 核心思想：协调优于复杂

每个智能体本身很简单 -- 只是调用 LLM 的函数。智能之处在于智能体的协调方式：它们看到什么上下文、何时运行、以及如何组合输出。

## 架构详解

### Agent（智能体）

智能体是 LLM 函数的薄包装。它有一个身份（名称、角色、目标），会注入系统提示词中，还有一个 `step()` 方法用于将任务和可选上下文发送给 LLM。

```python
agent = Agent(
    name="researcher",
    role="research analyst",
    goal="find accurate information",
    llm_fn=my_llm_function,
)
result = agent.step("什么是量子计算？")
```

### SharedMemory（共享内存）

一个简单的基于字典的键值存储。智能体将输出写入此处，其他智能体（或编排器）随后可以访问。这实现了智能体之间的松耦合。

```python
memory = SharedMemory()
memory.store("research", "量子计算使用量子比特。")
memory.retrieve("research")  # -> "量子计算使用量子比特。"
```

### Orchestrator（编排器）

编排器实现三种协调策略：

**顺序模式** -- 智能体依次运行。每个智能体接收原始任务和所有前序智能体的输出作为上下文。适用于后续阶段建立在前序阶段基础上的流水线（如研究 -> 分析 -> 总结）。

**并行模式** -- 所有智能体独立运行同一任务。没有任何智能体能看到其他智能体的输出。结果收集到列表中。适用于获取多样化视角（如多个独立分析）。

**辩论模式** -- 智能体在多轮中轮流优化共享答案。每个智能体看到当前最佳答案和完整辩论历史。最后一个智能体的响应成为新的最佳答案。适用于迭代优化和纠错。

```python
orch = Orchestrator(agents, memory)
result = orch.run_sequential("任务")  # 或 run_parallel, run_debate
```

### MultiAgentSystem（多智能体系统）

顶层类，将所有组件组合在一起。它内部创建共享内存和编排器，并根据配置的模式分派到正确的协调方法。

```python
system = MultiAgentSystem([agent_a, agent_b], mode="debate")
result = system.run("我们应该使用微服务吗？")
```

## 前向传播详解

### 顺序模式

```
任务: "分析 AI 对医疗保健的影响"

智能体 1（研究员）:
    输入:  任务（无上下文）
    输出:  "AI 改善诊断能力，实现个性化医疗..."

智能体 2（分析师）:
    输入:  任务 + "[研究员]: AI 改善诊断能力..."
    输出:  "关键优势：更快诊断、降低成本..."

智能体 3（评审员）:
    输入:  任务 + "[研究员]: ... \n[分析师]: ..."
    输出:  "关注点：数据隐私、训练数据偏差..."

最终输出: 智能体 3 的响应
```

### 并行模式

```
任务: "评估这个商业提案"

智能体 1（乐观者）: "强大的市场机会..."
智能体 2（悲观者）: "高风险因素..."
智能体 3（分析师）:  "中立评估..."

输出: ["强大市场...", "高风险...", "中立..."]
```

### 辩论模式

```
任务: "远程办公是否优于办公室办公？"

第 1 轮:
    智能体 A: "远程办公更好，因为..."
    智能体 B: "办公室办公更好，因为..."
    当前答案: 智能体 B 的响应

第 2 轮:
    智能体 A 看到当前答案 + 历史，精炼论点
    智能体 B 看到更新的答案 + 历史，精炼论点
    当前答案: 智能体 B 的更新响应

第 3 轮:
    同样的模式，进一步优化

最终输出: 所有轮次后的优化答案
```

## 练习

打开 `exercise.py` 来自己练习实现多智能体系统。

### 练习顺序

1. **Agent（智能体）**：实现创建和 `step()` 方法
2. **SharedMemory（共享内存）**：实现 store、retrieve、get_all、clear
3. **Orchestrator（编排器）**：实现顺序、并行和辩论模式
4. **MultiAgentSystem（多智能体系统）**：实现模式验证和分派

### 提示

- 对于 Agent，在用户消息之前始终构建包含角色和目标的系统提示词。
- 对于 SharedMemory，`retrieve()` 应对缺失的键抛出 `KeyError`，`get_all()` 应返回副本。
- 在顺序模式中，将上下文累积为带标签的字符串。
- 在辩论模式中，同时跟踪当前最佳答案和完整辩论历史。
- LLM 函数签名为 `fn(messages: list[dict]) -> str`，其中消息遵循 OpenAI 格式。

## 关键要点

1. **智能体是包装器，不是庞然大物。** 一个智能体只是一个 LLM 函数加上一个身份。真正的力量来自协调。

2. **协调模式至关重要。** 顺序模式用于流水线，并行模式用于多样性，辩论模式用于优化。选择正确的模式取决于任务。

3. **共享内存实现松耦合。** 智能体不直接相互通信 -- 它们读写共享内存。这使系统易于扩展。

4. **上下文决定一切。** 智能体看到的内容决定了它的输出。顺序模式提供丰富上下文；并行模式不提供；辩论模式提供迭代上下文。

5. **辩论提升质量。** 通过让智能体挑战和优化彼此的答案，辩论模式可以捕获错误并产生比任何单个智能体更平衡的结果。
